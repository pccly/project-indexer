# project-indexer

A Claude Code / Claude.ai skill that indexes your project into `.claude/PROJECT_INDEX.md` — a tree of every file with a one-line description — then wires it into `CLAUDE.md` via `@` import so future sessions understand the layout **without** running `Glob` / `Grep` / `find` every turn.

- **Full tree** of every tracked file (honors `.gitignore` + `.claudeignore`)
- **One-line descriptions** generated by reading each file
- **Two modes**: manual (on request) and auto (via PostToolUse hook)
- **Incremental updates** — only re-describes changed files (via git diff or mtime)
- **Idempotent `CLAUDE.md` integration** using marked `<!-- project-indexer:start -->` / `<!-- project-indexer:end -->` block

## Install

### Claude Code / Cursor / Windsurf / any agent (via `npx skills`)

```bash
npx skills add pccly/project-indexer            # auto-detect agent
npx skills add pccly/project-indexer -a cursor
npx skills add pccly/project-indexer -a windsurf
```

### Manual (any agent)

```bash
git clone https://github.com/pccly/project-indexer ~/.claude/skills/project-indexer
```

The skill lives at `~/.claude/skills/project-indexer/` for Claude Code. Other agents use their respective skills directory (e.g. `~/.agents/skills/` for Codex).

Uninstall: `npx skills remove project-indexer` or just `rm -rf` the directory.

## Usage

Once installed, trigger any of these phrases:

- "index this project"
- "update the project index"
- "map the codebase"
- "refresh PROJECT_INDEX.md"
- "rebuild the index from scratch"
- "enable auto indexing" / "disable auto indexing"

The skill will:
1. Scan the repo respecting `.gitignore` + `.claudeignore`
2. Read each file to generate a one-line description
3. Write `.claude/PROJECT_INDEX.md` with a full tree + per-file entries
4. Create or update `./CLAUDE.md` to `@`-import the index

### Slash commands (Claude Code)

If you copy the `commands/` directory to `~/.claude/commands/`, you also get:

- `/project-index` — create or incrementally update
- `/project-index-rebuild` — force full rebuild
- `/project-index-auto on|off` — toggle the auto-update hook

## Auto mode

Auto mode writes a PostToolUse hook to `.claude/settings.json` that regenerates the index in the background whenever `Write` or `Edit` tools run. The hook:

- **Debounces** — skips if the index was updated < 60 seconds ago
- **Skip-if-missing** — never auto-creates; only refreshes existing indexes
- **Non-blocking** — runs detached via `&`
- **Single-flight** — uses a lock file to prevent concurrent runs

Enable: ask the agent to "enable auto project indexing" or run `/project-index-auto on`.

Disable: `/project-index-auto off` or remove the hook block from `.claude/settings.json`.

## Example output

```markdown
# Project Index

_Last updated: 2026-04-24 (mode: incremental)_
_Root: /Users/you/code/my-app_
_Files indexed: 87_

## Tree

\`\`\`
src/
  main/
    ipc.ts
    index.ts
  renderer/
    App.tsx
package.json
\`\`\`

## Files

### `src/main/ipc.ts`
IPC handlers for workspaces, memories, skills, agents, MCP, plugins.

### `src/main/index.ts`
Electron main entry — creates BrowserWindow, wires native menu.

### `src/renderer/App.tsx`
Root React component — renders tool switcher, sidebar, main panel.
```

## Why

Every `Glob` / `Grep` call burns tokens and adds latency. One 200-line index replaces dozens of search calls for navigation tasks. The index also surfaces dead files, unclear naming, and coverage gaps every time you review it.
